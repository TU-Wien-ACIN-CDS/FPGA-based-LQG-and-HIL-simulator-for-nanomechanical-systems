Model Configuration tab

In this tab, you can configure and inspect up to two separate internal model configurations. Switch between the tabs to access parameters of each configuration.

The LQG implementation supports models with between 1 and 3 degrees of freedom (here dubbed X, Y, Z), with arbitrary, potentially full input and output matrices. You can also optionally enable a drift state.

For more details about the implemented model structure, please follow the Internal model reference page.

Parameter manipulation buttons

Push parameters button

Clicking this button recalculates the feedback and Kalman gains and pushes either the parameter Set 1 or 2 (depending on the position of the Parameter set select switch) to the hardware. Note: It is not necessary to push this button after switching either the Parameter set select or Enable feedback switches, as those execute the Push parameters callback.

Save parameters

This button summons a file-selection dialog to export the parameters into two separate .mat files (one for each set). Configured filename gets amended with the set number and a timestamp.

Load parameters

This button allows you to select a .mat parameter set and decide into which parameter set it is supposed to be uploaded.

Important: This function takes only the parameter files with the exact same structure as what the Save parameters button creates.

Import parameters

Contrary to the Load parameters function, this allows you to import parameters from a .json file. You can generate a template .json file by right-clicking the Import parameters button and clicking Generate template json.

Dynamic parameters panel

X/Y/Z Frequency

Natural (undamped) frequency of the respective degrees of freedom. Set the frequency to a negative value to get an unstable mechanical mode (negative spring stiffness). Unit: Hertz

Phi time constant

Time constant of the drift state. Unit: seconds

X/Y/Z Damping

Damping rate of the respective degrees of freedom. Unit: Hertz

X/Y/Z/Phi Process noise

Natural (undamped) frequency of the respective degrees of freedom. Unit: Hertz

Particle mass

Mass of the particle. Unit: kg

Detection + Actuation tab

Detection calibration coefficients

Fill in values in the X, Y, Z column for position measurements and, similarly, in the dX, dY, dZ column for velocity measurements. Leave a row empty (zero) if the analog input channel is unused.

Unit: V/m for X,Y,Z,Phi, V.s/m for dX,dY,dZ

Measurement noise variances

Fill in values of the measurement noise covariance. Note that the matrix should be symmetrical.

Unit: V^2/Hz

Actuator coupling coefficients

Fill in values in the X, Y, Z column for actuation acting in velocity and, similarly, in the dX, dY, dZ column for actuation acting in acceleration (e.g. electrodes acting on a particle). Leave a row empty (zero) if the analog output channel is unused.

Unit: N/V for X,Y,Z,Phi, V.s/m for dX,dY,dZ

Tuning variables tab

Use X/Y/Z/Phi check boxes

You can select which states are to be active for the state estimation. Note that this setting overrides any other configuration, e.g. the detection calibration.

Feedback-weighing matrix

Fill in the square-root values for the Feedback-weighing matrix (R matrix for the LQR). Unit: 1/V

Relative weighing of states

Fill in values in the state-weighing matrix (Q matrix for the LQR). Leave the matrix as eye matrix for the default energy weight. Note that the matrix should be symmetrical. Unit: unitless

Internal model format dropdown menu

Select which model transformation is to be used for the parameters set. Note that the transformed variants differ only numerically, but input-output behaviour is identical.

  • Standard - The standard representation, no transformation.
  • Transformed standard (recommended) - The standard representation "normalized" with natural frequency and the highest value in the detection calibration matrix.
  • Closed-loop - This transformation can be useful for dealing with unstable systems. Instead of the standard state-transition matrix (A-LC), a closed feedback loop is simulated (A-BK-LC). This matrix is simulated irrespective of the state of the Enable Feedback switch, that only enables the actual voltage output.
  • Transformed closed-loop (recommended for unstable systems) - Uses the combination of the closed loop and the normalization transformations, improving numerical stability over the non-transformed closed-loop variant.

Recalculate LQR button

Clicking this button parses all the parameters within the set and updates the output tables in the Feedback gain tab. Note that this operation does not update the hardware until the Push parameters button is clicked, so you can use it to tweak the feedback gain before actually applying the settings.

Feedback gain tab

K LQR matrix entries

You can inspect the resulting feedback gain matrix of the Recalculate LQG operation in this table. Note that the scale of the entries changes based on the internal model used!

Estimated output voltage standard deviation

Here you can inspect the estimated (triple) standard deviation of the output signal. Note that this estimate is calculated based on an ideal closed-loop system, hence it will be most probably optimistic. Unit: V

IO calibration tab

Output/Input calibration

Set the value of offset to be added to the signal (i.e. value of DAC1 offset of 0.020 raises the output voltage by 20 millivolts). Set the gain correction to make sure the internal signals (as plotted in the Data acquisition tab) correspond to the real physical values.

The internal calibrated input signal is calculated as calibratedInputSignal = (rawInputSignal + offset)*factor.

The calibrated output signal is calculated as calibratedOutputSignal = (rawOutputSignal*factor) + offset.

Advanced Tab

Warning: For most of the use-cases, you don't need to modify anything here! For more detail, follow the Numerical transformation section in the Internal model reference page.

Target state standard deviation

Set the target standard deviation of the estimated states. The LQG is rescaled such as to achieve this target with white noise with the variance given by the Measurement noise variance at the input.

Transform the state space

Checking this box enables rescaling of the state space model for better numerical behaviour. Unless you are diagnosing issues, or know exactly why you don't want this, keep this setting checked.

Limit the transformation

This option adds additional step to the transformation, tweaking the scaling such as none of the input- and output-matrix values do not overflow the available internal datatypes. Same as the option above, you probably want this option enabled.

Undo the transformation for data plotting and export

If checked, the GUI will undo the state-space scaling for plotting and data export, and the feedback matrix inspection, getting the values according to the Internal model format setting.
